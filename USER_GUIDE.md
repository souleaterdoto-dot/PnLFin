# Инструкция пользователя FinancePnL

FinancePnL - локальное desktop-приложение для загрузки сделок из Excel, ведения справочников курсов и ставок, расчета PnL и просмотра аналитики.

Приложение работает локально на компьютере пользователя. Данные хранятся в SQLite-базе в папке `FinancePnL_Data`.

## 1. Запуск приложения

### Обычный запуск

1. Откройте файл `FinancePnL.exe`.
2. Дождитесь открытия главного окна.
3. При первом запуске приложение автоматически создаст папку:

```text
FinancePnL_Data/
```

И файл базы:

```text
FinancePnL_Data/app.sqlite3
```

### Важный нюанс по базе

Приложение рассчитано на работу одного пользователя с одной локальной базой в конкретный момент времени.

Если второй пользователь или второй экземпляр приложения попробует подключиться к этой же базе, приложение должно заблокировать запуск, чтобы избежать конфликтов данных.

## 2. Основные разделы

В левом меню доступны основные разделы:

- `Сделки`
- `Выгрузить из МП`
- `Курсы`
- `Ставки`
- `Аналитика`
- `Системное`

## 3. Раздел "Выгрузить из МП"

Этот раздел используется для загрузки Excel-файла со сделками.

### Как должен быть подготовлен Excel

Приложение читает только готовый результат из Excel. Оно не обновляет Power Query, не изменяет файл и не ломает подключения.

Поддерживается логика:

1. Сначала ищется Excel Table с именем `All`.
2. Если таблицы нет, ищется лист `All`.
3. Если листа `All` нет, приложение предлагает выбрать лист.

Перед загрузкой пользователь должен сам открыть Excel, обновить Power Query, сохранить файл и только потом загрузить его в приложение.

### Что происходит при импорте

1. Пользователь выбирает Excel-файл.
2. Приложение открывает его в режиме read-only.
3. Проверяет наличие обязательных колонок.
4. Валидирует строки.
5. Пустые строки пропускает.
6. Ошибочные строки складывает в ошибки импорта.
7. Успешные строки сохраняет в реестр сделок.

### Важный нюанс

При новом импорте реестр сделок полностью заменяется новыми данными. Это сделано специально, чтобы данные не накапливались дублями.

### Цвета и типы сделок

Тип сделки определяется по валюте и знаку суммы:

- положительная сумма - импорт;
- отрицательная сумма - экспорт, но в интерфейсе сумма отображается положительно;
- `USDT` выделяется отдельно как `USDT импорт` или `USDT экспорт`.

## 4. Раздел "Сделки"

Это основной рабочий реестр.

### Режимы отображения

Реестр большой, поэтому данные разделены на режимы:

- `Общий`
- `Курсы`
- `Финансы USD`
- `Финансы валюта`
- `PnL`
- `Все`

Первые базовые поля присутствуют во всех режимах:

- номер сделки;
- название клиента;
- дата фиксации с клиентом.

### Пагинация

Внизу реестра можно выбрать количество строк на странице:

- 25;
- 50;
- 100.

Также можно ввести конкретный номер страницы и перейти к ней.

### Поиск и фильтры

В реестре доступны:

- общий поиск;
- фильтры по колонкам;
- сортировка от А до Я и от Я до А;
- фильтр по субагентам;
- фильтр по рефералам;
- фильтр по типу операции;
- фильтры по маркерам проверки.

Фильтры в шапке работают похожим образом на Excel: можно раскрыть список значений, выбрать нужные и применить.

### Маркеры проверки

У клиента в строке можно поставить маркер:

- проверено;
- под вопросом.

Маркер сохраняется в базе и отображается рядом с названием клиента.

Эти маркеры нужны для визуального отсеивания строк.

### Редактирование сделки

Чтобы открыть карточку сделки:

1. Дважды нажмите по строке.
2. Откроется детальная модалка.
3. Измените нужные данные.
4. Нажмите сохранить.

Карточка разделена на вкладки, чтобы не перегружать форму.

### Добавление сделки вручную

В реестре есть кнопка добавления строки.

Обязательные поля для ручного добавления:

- менеджер;
- дата поступления заявки;
- дата фиксации с клиентом;
- дата списания с баланса ПА;
- название клиента;
- компания получатель;
- страна банка получателя;
- все поля во вкладке курсов.

Финансовые поля можно оставить пустыми.

Если реферал не заполнен, используется значение `Без банка`.

### Удаление сделки

Для удаления:

1. Выберите строку.
2. Нажмите удалить.
3. Подтвердите действие.

Удаление необратимо для этой локальной базы.

## 5. Курсы и кросс-курсы

Курсы используются для пересчета сумм в USD и расчета PnL.

### Раздел "Курсы"

В разделе курсов отображается календарь.

Дни, по которым есть курсы, отмечаются визуально. При открытии дня можно посмотреть курсы, добавить ручной курс или подтянуть курс из ЦБ.

### Ручные курсы

Если ручной курс существует, он имеет приоритет над курсом ЦБ.

### Курсы ЦБ

Курсы ЦБ подтягиваются локально через сервис приложения.

Для `CNH` используется курс `CNY`, потому что в ЦБ обычно есть `CNY`, а сделки могут вестись в `CNH`.

### Нет курса

Если по сделке не найден курс, рядом с проблемным значением появляется индикатор.

По индикатору можно открыть модалку и попробовать подтянуть курс за нужную дату.

Если валюты нет в ЦБ, приложение должно явно показать, что для этой валюты курс в ЦБ не найден, и при возможности предложить похожие варианты.

## 6. Раздел "Ставки"

Раздел `Ставки` используется для ведения рефералов и условий ставок.

### Рефералы

На главной странице ставок отображаются карточки рефералов.

Рефералы создаются на основе значений из реестра сделок и сохраняются в базе.

Если при новом импорте в сделках появился новый реферал, он создается в справочнике.

Если реферал неактивен, ставка по нему считается равной 0.

### Условия ставок

У каждого реферала есть условия ставок.

Важно: условия ставок описывают наши комиссии по условиям реферала, а не готовую сумму `Ставка реферала, USD`.

Итоговая `Ставка реферала, USD` считается так:

```text
Ставка реферала, USD =
комиссия клиента % в USD
+ фикс комиссия клиента в USD
- процентная комиссия из условий в USD
- фиксированная комиссия из условий в USD
```

То есть приложение сравнивает, сколько взяли с клиента по проценту и фиксу, и сколько должно остаться по условию. Разница попадает в колонку `Ставка реферала, USD`.

Условия разделены по типам:

- процентные ставки от эквивалента USD;
- процентные ставки в валюте сделки;
- фиксированные комиссии от эквивалента USD;
- фиксированные комиссии в валюте сделки.

В условии задаются:

- активность;
- валюта;
- сумма от;
- сумма до;
- дата начала;
- дата окончания;
- процент;
- фиксированная сумма;
- валюта фиксированной суммы;
- комментарий.

Пустая дата окончания означает бессрочное условие.

### Нахлест условий

Для одного реферала не должно быть двух активных условий, которые подходят под одну и ту же сделку.

Приложение проверяет конфликты и не дает сохранить условие, если оно пересекается с уже существующим.

### Импорт условий из Excel

Можно загрузить Excel с условиями ставок.

Формат:

```text
A1: eq. или В валюте сделки

Реферал
Тип сделки
Валюта
От
До
Процент, %
Фиксированная сумма
Дата начала
Дата окончания
```

Если в `A1` указано `eq.`, условия попадут в блок ставок от эквивалента USD.

Если в `A1` указано `В валюте сделки`, условия попадут в блок ставок в валюте сделки.

Важно: импорт условий не создает новых рефералов. Если реферала нет в справочнике, строка считается некорректной.

### Пример файла

В разделе ставок есть кнопка `Показать пример`. Она создает и открывает пример Excel-файла нужного формата.

## 7. Клиенты-исключения

В разделе ставок есть отдельная вкладка для клиентов-исключений.

Исключение используется, когда для конкретного клиента в определенный период ставку нужно задавать вручную.

В исключении указываются:

- клиент;
- комментарий или условие;
- дата начала;
- дата окончания.

Пустая дата окончания означает бессрочно.

Если сделка попадает под исключение, рядом со ставкой реферала появляется индикатор. По нему можно открыть форму и ввести ручную ставку.

## 8. Расчет PnL

Главный расчет выполняется в режиме `PnL`.

Формула:

```text
PnL =
(комиссия клиента % + фикс комиссия USD + SWIFT USD + штраф за переотправку)
-
(комиссия ПА USD + комиссия за SWIFT ПА USD + ставка реферала USD)
```

### Что входит в доходы

- комиссия клиента по проценту;
- фиксированная комиссия;
- SWIFT;
- штраф за переотправку.

### Что входит в расходы

- комиссия платежного агента;
- комиссия за SWIFT платежного агента;
- комиссия реферала.

### Экспортные сделки

В Excel экспортные сделки могут приходить с отрицательной суммой и отрицательными комиссиями.

В расчетах приложение приводит такие значения к положительным, чтобы PnL считался корректно.

### Возвраты

Если заполнена дата возврата средств клиенту:

- общая ставка клиента становится 0;
- фикс комиссия становится 0;
- SWIFT становится 0;
- комиссия реферала становится 0.

### Повторный платеж

Если у сделки стоит признак повторного платежа, рядом с номером сделки отображается индикатор.

По индикатору можно открыть форму и ввести:

- процент комиссии за переотправку;
- штраф в USD.

Комиссия за переотправку прибавляется к ставке клиента.

Штраф за переотправку увеличивает PnL, так как его платит клиент.

Для повторной переотправки комиссия реферала считается 0.

### Ручные значения PnL

В карточке сделки есть вкладка `Системное / PnL`.

Там пользователь может вручную задать значения в USD.

Если значение задано вручную, в таблице отображается маркер, чтобы было видно, что сумма не рассчитана автоматически.

Также есть кнопка, которая позволяет подтянуть значения из базы/расчета обратно.

## 9. Аналитика

Раздел `Аналитика` работает через локальный Dash/Plotly dashboard.

### Как это устроено

Flet-приложение остается основным desktop-интерфейсом.

Dash запускается локально на:

```text
http://127.0.0.1:<порт>
```

Обычно порт начинается с `8050`. Если он занят, приложение ищет следующий свободный порт.

Dash только читает данные из SQLite и не изменяет базу.

### Важный нюанс для RDP

Аналитика откроется только там, где запущено приложение.

Если приложение запущено на RDP-сервере, браузер тоже должен открываться в этой же RDP-сессии.

Если ссылка вместо `127.0.0.1` превращается в имя сервера, например `STORAGE-01...`, это проблема окружения или перенаправления URL. Корректная ссылка должна быть именно:

```text
http://127.0.0.1:<порт>
```

### Вкладки аналитики

В аналитике есть вкладки:

- `Рефералы`;
- `Клиенты`;
- `Партнеры`;
- `Обороты`;
- `Рентабельность`.

### Глобальные фильтры

Вверху аналитики доступны общие фильтры:

- тип операции;
- режим графика;
- валюта отображения;
- вид расчета.

Типы операций:

- импорт;
- экспорт;
- USDT импорт;
- USDT экспорт.

### Рефералы

Показывает PnL по каждому рефералу.

Можно смотреть:

- в USD;
- в валюте сделки;
- по доходам/расходам;
- как итоговый PnL.

### Клиенты

Во вкладке клиентов отображаются топ-клиенты по выбранному рефералу.

Показываются первые 9 клиентов и отдельная карточка `Остальные`.

При нажатии на клиента открывается детальный график по нему.

### Партнеры

Показывает PnL в разрезе платежных агентов/партнеров.

### Обороты

Показывает обороты:

- в валюте сделки;
- в USD.

### Рентабельность

Показывает рентабельность по:

- рефералам;
- партнерам.

## 10. Системное

Раздел `Системное` используется для технических и ручных настроек.

В текущей логике особенно важна вкладка PnL в карточке сделки, где можно вручную задать значения в USD для конкретной строки.

## 11. Частые ситуации

### Не видно картинок рефералов

Проверьте, что папка `assets/referrals` попала в сборку или находится рядом с исходниками при запуске из Python.

В EXE ассеты должны быть упакованы через `FinancePnL.spec`.

### Аналитика не открылась

Проверьте:

- не заблокирован ли локальный порт;
- не запрещает ли политика RDP открытие локального браузера;
- ссылка должна быть `http://127.0.0.1:<порт>`;
- приложение должно быть запущено на той же машине, где открывается браузер.

### Excel не импортируется

Проверьте:

- есть ли лист или таблица `All`;
- сохранен ли файл после обновления Power Query;
- есть ли обязательные колонки;
- не открыт ли файл в режиме, который блокирует чтение;
- корректны ли даты и числа.

### Сделка показывает "Нет курса"

Значит приложение не смогло рассчитать USD-эквивалент.

Нажмите на индикатор рядом с проблемным полем и попробуйте подтянуть курс за дату сделки.

Если валюты нет в ЦБ, курс нужно будет добавить вручную или выбрать похожий вариант, если приложение его предложит.

### Ставка реферала "Нет ставки"

Возможные причины:

- реферал не найден;
- реферал неактивен;
- нет активного условия;
- сумма не попадает в диапазон;
- дата сделки вне периода условия;
- валюта не совпадает.

Если реферал `Без банка`, ставка считается 0.

Если реферал неактивен, ставка тоже считается 0.

### Появился конфликт условий

Это значит, что два активных условия одного реферала могут подойти под одну и ту же сделку.

Нужно изменить диапазон суммы, валюту, даты или отключить одно из условий.

## 12. Что нельзя делать

Не редактируйте файл `FinancePnL_Data/app.sqlite3` вручную.

Не запускайте несколько экземпляров приложения на одной базе.

Не удаляйте папку `FinancePnL_Data`, если хотите сохранить данные.

Не ожидайте, что приложение само обновит Power Query в Excel. Power Query обновляется вручную в Excel до импорта.

## 13. Резервное копирование

Для резервной копии достаточно сохранить папку:

```text
FinancePnL_Data/
```

Главный файл данных:

```text
FinancePnL_Data/app.sqlite3
```

Перед копированием лучше закрыть приложение.

## 14. Короткий рабочий сценарий

1. Обновить Power Query в Excel.
2. Сохранить Excel.
3. Открыть FinancePnL.
4. Перейти в `Выгрузить из МП`.
5. Загрузить Excel.
6. Проверить ошибки импорта.
7. Перейти в `Сделки`.
8. Проверить курсы, ставки и проблемные индикаторы.
9. При необходимости заполнить справочник ставок.
10. Проверить режим `PnL`.
11. Открыть `Аналитика` и посмотреть разрезы по рефералам, партнерам, клиентам, оборотам и рентабельности.
